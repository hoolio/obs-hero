Structure

Direction C's hero, unchanged — one question, one sentence, one live chart. The exploration happens on the way down: five bands, each a “see your ___”, each with a real interactive notebook and an explicit TRY IT instruction so nobody scrolls past a chart without touching it.

App feel

Scroll-progress bar, a right-hand rail that tracks which band you're in, and sticky headline columns so the text holds still while the chart moves. Bands alternate background so each one reads as its own screen.

Contentful shape

One Hero entry plus a repeatable SeeBand: label, number, headline, body, tryIt, notebookSlug, cells, stageHeight. Editors reorder, add, or retire a band without touching code — that's the whole point of this structure.

Blocked — band 03

@mpkhinda/nyc-city-pay is the right notebook, but its main Plot.plot() cell is unnamed, so ?cells= can't target it and this band renders empty today. Embedding the whole notebook instead dumps the title, source notes, key variables and every helper-function signature — the exact clutter we're removing. Fix is one step: fork it, name the cell chart, keep Matt Khinda's credit. The URL here already assumes that name and will light up the moment it exists.

Known cost

Six live iframes on one page. Needs lazy-loading below the fold and a static poster image per band before this goes anywhere near production.

Also unresolved

Observable's embeds render their own footer bar inside the frame; I clip it with an overflow window. The clean fix is the Observable runtime instead of iframes — more work, less copy-pasteable into Contentful. Still worth deciding.
