Why people use Observable

It feels instant, and everything you need is already there.

Fast & lightweight

Just start writing

A web-based environment with batteries included. No dev setup, no project to init — and every edit runs instantly, like a spreadsheet. Hot reloading on steroids.

Batteries included

Good opinions, built in

The libraries, the data connectors, the primitives — the things you need are already there. D3 and Plot ship by default.

Connects to everything

Files, databases, APIs

Polyglot by design — JavaScript, SQL, Markdown, HTML side by side. Self-serve, so anyone can explore without a ticket to the data team.

Built-in collaboration

Share instantly

Real-time or async, with permission controls, sandboxing, and version history. Follow along and be inspired by your team and the community.

Trustworthy

Inspectable and auditable

Every result is transparent — hover any value, trace any number to its source. Not a black box.

Built-in agent

An AI that sees your data

The integrated agent runs code and inspects live values — it sees how data is distributed and what a chart actually rendered. Grounded, not guessing.

Instead of local development, a chat window, Hex, Tableau, or hiring a data analyst — one place that spans all of it.

Plug into open data

Proposal notes

The reframe

This is the “do / why” direction: the campaign leads with a question you ask, not a chart you watch. The hero is an Ask box; clicking a public-data prompt runs the agent and swaps in a real live notebook answering it.

Real data, real embeds

All four demo answers are genuine public-data notebooks (BLS county unemployment, WHO life expectancy, NYC payroll, US Census). The agent's SQL is illustrative; the rendered chart is live.

Structure

Hero ask → the Ask→Ship spectrum (with the Build actions) → Why Observable (fast, batteries, collaboration, connects, trustworthy, agent, plus the “versus” line) → open-data sources → CTA.
